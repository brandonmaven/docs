The widget renders the same way for every session unless you customize it. You can save defaults **per-project** in your dashboard and/or override **per-session** in the API call.

## Where customization lives

<CardGroup cols={2}>
  <Card title="Project defaults (recommended)" icon="palette">
    Set once in **Dashboard → your app → Chat Payments tab → Customize**. Applies to every session automatically.
  </Card>
  <Card title="Per-session overrides" icon="settings-2">
    Pass `theme`, `labels`, or `fields` in the `POST /v1/widget-sessions` body to override project defaults for one session.
  </Card>
</CardGroup>

**Merge order:** per-session body > project defaults > built-in defaults.

## Theme

Controls colors, typography, and inner card sizing.

```json
{
  "theme": {
    "mode": "light",
    "primary": "#6366F1",
    "primary_text": "#ffffff",
    "background": "#ffffff",
    "text": "#111214",
    "muted": "#6b6f76",
    "border": "#e6e6ea",
    "radius": "8px",
    "card_max_width": "420px",
    "card_padding": "20px",
    "font_size": "14px",
    "font_family": "\"Inter\", -apple-system, sans-serif",
    "font_url": "https://fonts.googleapis.com/css2?family=Inter"
  }
}
```

| Field | Example | Description |
|---|---|---|
| `mode` | `"light"` \| `"dark"` | Flips the default palette. Merchant overrides still win. |
| `primary` | `"#6366F1"` | Pay button background color. |
| `primary_text` | `"#ffffff"` | Pay button text color. |
| `background` | `"#ffffff"` | Card background. |
| `text` | `"#111214"` | Primary text color. |
| `muted` | `"#6b6f76"` | Secondary text (deposit line, labels). |
| `border` | `"#e6e6ea"` | Card + input border color. |
| `radius` | `"4px"` · `"8px"` · `"14px"` · `"20px"` | Input + button border radius. |
| `card_max_width` | `"320px"`–`"560px"` \| `"100%"` | Max width of the card. Container controls outer width. |
| `card_padding` | `"14px"` · `"20px"` · `"28px"` | Internal card padding. |
| `font_size` | `"13px"`–`"16px"` | Base font size. |
| `font_family` | CSS font stack | Override the default font. |
| `font_url` | Stylesheet URL | Optional — load a web font (e.g. Google Fonts) before applying. |

### Dark mode

Setting `mode: "dark"` flips the full palette to dark-theme-appropriate values:
- Dark background, light text
- Field backgrounds darken
- Error/success boxes shift to translucent color washes

You can still override individual colors — they take precedence over the dark defaults.

## Labels

Customize visible text.

```json
{
  "labels": {
    "pay_button": "Confirm and Pay",
    "processing": "Processing",
    "merchant_name": "Secure Payment",
    "deposit": "You will be charged $100 today as a deposit.",
    "success_title": "Payment confirmed",
    "success_detail": "Your card was charged successfully.",
    "failure_title": "Payment Failed"
  }
}
```

| Field | Default | Where it shows |
|---|---|---|
| `merchant_name` | `"Secure Payment"` | Header title next to the lock icon |
| `pay_button` | `"Confirm and Pay"` | The green submit button |
| `processing` | `"Processing"` | Button label while charging |
| `deposit` | Auto-generated from amount | Small text above the button |
| `success_title` | `"Payment confirmed"` | Green checkmark screen title |
| `success_detail` | `"Your card was charged successfully."` | Green checkmark body text |
| `failure_title` | `"Payment Failed"` | Red error box title |

## Fields

Show/hide optional fields.

```json
{
  "fields": {
    "cardholder_name": true,
    "billing_zip": true
  }
}
```

| Field | Default | Notes |
|---|---|---|
| `cardholder_name` | `true` | When shown, the name is sent to the gateway (Braintree/Authnet `firstName/lastName`, Stripe `billing_details.name`, Shift4 `cardholderName`). |
| `billing_zip` | `true` | Sent to the gateway for AVS (address verification). Some issuers require it. |

Card number, expiry, and CVV are always shown and required.

## Sizing for chatbots

The iframe fills 100% of its container — **width is controlled by your chatbot's CSS**, not by Maven. Inside the iframe, the card is centered and has a `max-width` (default 420px) you can control via `theme.card_max_width`.

## Live updates

After mounting, you can update theme/labels/fields at runtime — useful if your app has a light/dark toggle or the amount changes:

```javascript
const payment = Maven.createPayment({ sessionId }).mount("#slot");

// Later:
payment.updateTheme({ mode: "dark" });
payment.updateLabels({ pay_button: "Pay $49.99 now" });
payment.updateFields({ billing_zip: false });
```

## What you can't customize

Intentionally locked down:
- **Field heights, input spacing, field order** — brand-consistent across Maven so the form always feels usable
- **Minimum card size** — the form needs a floor to avoid overlap
- **Custom CSS injection** — disabled for security (opens XSS into the PCI iframe)

If a styling knob you want isn't on this page, let us know at [support@trymaven.com](mailto:support@trymaven.com).
