# Retell Integration

Add PCI-compliant voice payments to your Retell AI agent. When your agent needs to collect a payment, it calls Maven to create a session and transfers the caller to Maven's secure payment line.

## How It Works

<Steps>
  <Step title="Agent triggers collect_payment">
    During a call, your Retell agent calls the `collect_payment` custom function with the amount and caller's phone number.
  </Step>
  <Step title="Maven creates a session">
    Maven creates a payment session and returns a phone number to transfer the caller to.
  </Step>
  <Step title="Agent transfers the caller">
    Your agent uses Retell's `transfer_call` function to send the caller to Maven's payment line.
  </Step>
  <Step title="Payment collected">
    Maven collects the card details, processes the payment, and sends a [webhook](/integrations/webhooks) with the result. The caller is optionally transferred back to your agent via the `callback` number.
  </Step>
</Steps>

## Setup

Retell requires manual configuration of custom functions. Here's how to set it up:

<Steps>
  <Step title="Get your webhook token">
    In the [Maven Dashboard](https://app.trymaven.com), go to your app's **Integrations** tab and generate a Retell webhook token. This gives you a URL like:
    ```
    https://api.trymaven.com/integrations/retell/webhook?token=whk_xxx
    ```
  </Step>
  <Step title="Create the collect_payment function in Retell">
    In your Retell agent, add a **Custom Function** with:
    - **Name**: `collect_payment`
    - **URL**: Your webhook URL from step 1
    - **Description**: "Collect a payment from the caller. Call this when the caller is ready to pay."
    - **Parameters**:
      - `amount` (number, required) — Payment amount in dollars
      - `description` (string, optional) — What the payment is for
      - `callback` (string, optional) — Phone number to transfer back to after payment
      - `memory` (string, optional) — Summary of the call so far

    You do **not** need to add `caller` — Maven auto-fills it from `call.from_number`.
  </Step>
  <Step title="Create the transfer_call function in Retell">
    Add another Custom Function:
    - **Name**: `transfer_call`
    - **Description**: "Transfer the caller to the payment line. Call this after collect_payment returns a transfer_number."
    - **Parameters**:
      - `transfer_number` (string, required) — The phone number returned by collect_payment
  </Step>
  <Step title="Add get_session and cancel_session (optional)">
    You can also add:
    - **`get_session`** — Look up a session by phone number. Parameter: `phone_number` (string, required)
    - **`cancel_session`** — Cancel a pending session. Parameter: `session_id` (string, required)

    Both use the same webhook URL.
  </Step>
</Steps>

<Tip>
**Prompt your agent** to call `collect_payment` first, then use `transfer_call` with the returned `transfer_number`. Example system prompt addition: *"When the customer is ready to pay, call collect_payment with the amount. Then immediately call transfer_call with the transfer_number from the response."*
</Tip>

## Retell Payload Format

Retell sends custom function calls as:

```json
{
  "name": "collect_payment",
  "args": {
    "amount": 49.99,
    "description": "Monthly subscription"
  },
  "call": {
    "call_id": "retell_call_xxx",
    "from_number": "+14155551234",
    "metadata": {}
  }
}
```

The `caller` is auto-filled from `call.from_number` if not provided in `args`.

## Function Reference

### collect_payment

Creates a payment session. Returns a phone number to transfer the caller to.

**Parameters (in `args`):**

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `amount` | number | Yes | Payment amount in dollars (e.g., `150.00`) |
| `caller` | string | No | Customer phone number in E.164 format. Auto-filled from `call.from_number` if omitted. |
| `description` | string | No | Description of what the payment is for |
| `callback` | string | No | Phone number to transfer the caller back to after payment |
| `memory` | string | No | Summary of the call context to carry across the transfer |

**Response:**

```json
{
  "session_id": "a1b2c3d4-...",
  "transfer_number": "+18005551234",
  "message": "Payment session created. Transfer the caller to +18005551234 to securely collect their card details."
}
```

### get_session

Looks up the most recent payment session for a caller.

**Parameters (in `args`):**

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `phone_number` | string | Yes | Customer phone number in E.164 format |

**Response:**

```json
{
  "session_id": "a1b2c3d4-...",
  "status": "payment-success",
  "amount": 49.99,
  "currency": "USD",
  "mode": "charge",
  "gateway": "stripe",
  "caller": "+14155551234",
  "description": "Monthly subscription",
  "error": null,
  "created_at": "2026-03-07T10:09:29Z",
  "completed_at": "2026-03-07T10:10:30Z"
}
```

### cancel_session

Cancels a pending payment session.

**Parameters (in `args`):**

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `session_id` | string | Yes | The session ID returned by `collect_payment` |

**Response:**

```json
{
  "status": "cancelled",
  "session_id": "a1b2c3d4-..."
}
```

## Session Statuses

| Status | Description |
|--------|-------------|
| `created` | Session created, awaiting caller |
| `collecting-card` | Caller is entering their card number |
| `collecting-expiry` | Caller is entering expiration date |
| `collecting-cvv` | Caller is entering CVV |
| `collecting-postal-code` | Caller is entering postal code (if required) |
| `processing` | Card submitted to payment gateway |
| `payment-success` | Payment charged or card tokenized |
| `payment-failed` | Payment declined or error |
| `expired` | Session TTL exceeded |
| `cancelled` | Cancelled by API |
