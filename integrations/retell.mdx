# Retell Integration

Add PCI-compliant voice payments to your Retell AI agent. When your agent needs to collect a payment, it calls Maven to create a session and transfers the caller to Maven's secure payment line.

## How It Works

<Steps>
  <Step title="Agent triggers collect_payment">
    During a call, your Retell agent calls the `collect_payment` custom function with the amount and caller's phone number.
  </Step>
  <Step title="Maven creates a session">
    Maven creates a payment session and returns a phone number (or SIP URI) to transfer the caller to.
  </Step>
  <Step title="Agent transfers the caller">
    Your agent transfers the caller to the returned number.
  </Step>
  <Step title="Payment collected">
    Maven collects the card details, processes the payment, and sends a [webhook](/integrations/webhooks) with the result. The caller is optionally transferred back to your agent via the `callback` number.
  </Step>
</Steps>

## Setup

### Webhook URL

Configure a custom function in your Retell agent that calls Maven's webhook endpoint:

```
https://pciapi.trymaven.com/integrations/retell/webhook?token=YOUR_TOKEN
```

Generate a webhook token from the [Maven Dashboard](https://prod.trymaven.com) in your app's **Integrations** tab. The token encodes your API key, project, gateway, and mode.

### Retell Payload Format

Retell sends custom function calls as:

```json
{
  "name": "collect_payment",
  "args": {
    "amount": 49.99,
    "caller": "+14155551234",
    "description": "Monthly subscription"
  },
  "call": {
    "call_id": "retell_call_xxx",
    "from_number": "+14155551234",
    "metadata": {}
  }
}
```

The `caller` is auto-filled from `call.from_number` if not provided in `args`.

## Function Reference

### collect_payment

Creates a payment session. Returns a phone number to transfer the caller to.

**Parameters (in `args`):**

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `amount` | number | Yes | Payment amount in dollars (e.g., `150.00`) |
| `caller` | string | No | Customer phone number in E.164 format. Auto-filled from `call.from_number` if omitted. |
| `description` | string | No | Description of what the payment is for |
| `callback` | string | No | Phone number to transfer the caller back to after payment |
| `context` | string | No | Summary of the call context to carry across the transfer |

**Response:**

```json
{
  "session_id": "a1b2c3d4-...",
  "transfer_number": "+18005551234",
  "sip_uri": "sip:...",
  "message": "Payment session created. Transfer the caller to +18005551234 to securely collect their card details."
}
```

### get_session

Looks up the most recent payment session for a caller.

**Parameters (in `args`):**

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `phone_number` | string | Yes | Customer phone number in E.164 format |

**Response:**

```json
{
  "session_id": "a1b2c3d4-...",
  "status": "payment-success",
  "amount": 49.99,
  "currency": "USD",
  "mode": "charge",
  "gateway": "stripe",
  "caller": "+14155551234"
}
```

### cancel_session

Cancels a pending payment session.

**Parameters (in `args`):**

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `session_id` | string | Yes | The session ID returned by `collect_payment` |

**Response:**

```json
{
  "status": "cancelled",
  "session_id": "a1b2c3d4-..."
}
```
