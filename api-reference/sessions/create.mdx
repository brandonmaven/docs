---
title: "Create Session"
api: "POST /v1/sessions"
description: "Create a new payment session"
---

Create a new payment session. Sessions represent a single payment collection attempt. Create one when your AI agent needs to collect a payment or save a card.

<Note>
  Creating a new session for a phone number that already has an active session will automatically cancel the previous session.
</Note>

## Request Body

### Required

<ParamField body="project" type="string" required>
  Project identifier (UUID or slug).
</ParamField>

<ParamField body="caller" type="string" required>
  Customer's phone number in E.164 format (e.g., `+14155551234`).
</ParamField>

<ParamField body="amount" type="number" required>
  Amount to charge in cents (e.g., `2500` for $25.00). Must be greater than 0.
</ParamField>

<ParamField body="mode" type="string" required>
  Payment mode: `charge` (immediate payment) or `tokenize` (save card for later).
</ParamField>

### Optional

<ParamField body="input_mode" type="string" default="dtmf">
  How the customer enters card info during the call: `dtmf` (keypad) or `asr` (speech recognition). Only applies to `voice` channel.
</ParamField>

<ParamField body="email" type="string">
  Customer's email address. Used for receipt delivery if enabled in project settings.
</ParamField>

<ParamField body="full_name" type="string">
  Customer's full name (max 255 chars).
</ParamField>

<ParamField body="customer_id" type="string">
  Your internal customer identifier (max 255 chars). Returned in API responses for correlation.
</ParamField>

<ParamField body="description" type="string">
  Description shown on the customer's card statement (max 500 chars).
</ParamField>

<ParamField body="callback" type="string">
  Phone number to transfer the customer back to after payment (E.164 format, e.g., `+18005551234`).
</ParamField>

<ParamField body="memory" type="string">
  Context to share with the receiving agent on transfer-back (max 2000 chars).
</ParamField>

<ParamField body="metadata" type="object">
  Custom key-value data attached to the session. Returned in API responses.
</ParamField>

## Response

<ResponseField name="session_id" type="string">
  Unique session identifier (UUID).
</ResponseField>

<ResponseField name="status" type="string">
  Initial session status: `created`.
</ResponseField>

<ResponseField name="phone_number" type="string">
  Maven phone number for the customer to call or be transferred to.
</ResponseField>

<ResponseField name="created_at" type="string">
  ISO 8601 timestamp.
</ResponseField>
